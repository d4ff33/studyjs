1. Values, Types, and Operators

  • Arithmetic Operators: Symbols used for mathematical calculations, such as addition (+), subtraction (-), multiplication (*), division (/), and remainder (%).
  • Automatic Type Conversion (Type Coercion): JavaScript's behavior of automatically converting a value from one type to another when an operator requires a different type (e.g., converting "5" to the number 5 in "5" - 1). Can lead to unexpected results.
  • Binary Operator: An operator that acts on two values (e.g., + in 3 + 4, > in x > y).
  • Bits: The fundamental unit of data in a computer, representing a two-valued state (usually 0 or 1).
  • Boolean: A data type that has only two possible values: true and false. Often the result of comparisons.
  • Comparison Operators: Operators used to compare values, resulting in a Boolean (true or false). Includes >, <, >=, <=, == (equal to), != (not equal to), === (strictly equal to), !== (strictly not equal to).
  • Concatenation: The operation of joining strings together, typically done using the + operator.
  • Conditional Operator (Ternary Operator): The ? : operator. It takes three values: a condition, a value to return if the condition is true, and a value to return if the condition is false (condition ? value_if_true : value_if_false).
  • Empty Values: The special values null and undefined, used to signify the absence of a meaningful value.
  • Escaping: Using a backslash (\) before a character in a string literal to give it a special meaning (e.g., \n for newline, \" for a literal quote) or to negate its special meaning.
  • Expression: A fragment of code that produces a value.
  • Infinity / -Infinity: Special number values representing positive and negative infinity, resulting from operations like dividing by zero.
  • Logical Operators: Operators that work with Boolean values: && (logical AND), || (logical OR), and ! (logical NOT).
  • Modulo Operator (%): An arithmetic operator that returns the remainder of a division operation.
  • NaN: A special numeric value meaning "Not a Number". It results from mathematically nonsensical operations (e.g., 0 / 0, "five" * 2). NaN is the only value not equal to itself.
  • null: A primitive value representing the intentional absence of any object value.
  • Nullish Coalescing Operator (??): A logical operator that returns its right-hand side operand when its left-hand side operand is null or undefined, and otherwise returns its left-hand side operand.
  • Number: A data type representing numeric values, including integers and fractional numbers. JavaScript uses 64-bit floating-point representation.
  • Operator: A symbol or keyword that performs an operation on one or more values (operands). Examples: +, *, typeof, &&.
  • Operator Precedence: The rules that determine the order in which operators are applied in an expression when parentheses are not used (e.g., multiplication * happens before addition +).
  • Short-Circuit Evaluation: The behavior of logical operators (&&, ||, ??) where the right-hand operand is evaluated only if necessary to determine the result.
  • String: A data type used to represent text. String literals are enclosed in single quotes (' '), double quotes (" "), or backticks (` `).
  • Strict Equality/Inequality (===, !==): Comparison operators that check for equality or inequality without performing automatic type conversion. Generally preferred over == and !=.
  • Template Literals: Strings created using backticks (` `). They allow for multi-line strings and embedding expressions using the ${expression} syntax.
  • Ternary Operator: See Conditional Operator.
  • Type: The category that defines the role and capabilities of a value (e.g., number, string, boolean).
  • typeof Operator: A unary operator that returns a string indicating the data type of its operand.
  • Unary Operator: An operator that acts on a single value (e.g., - in -5, ! in !true, typeof).
  • undefined: A primitive value automatically assigned to variables that have just been declared, or representing the absence of a value in various other situations.
  • Unicode: A standard for encoding characters from virtually all writing systems, assigning a unique number to each character. JavaScript uses this standard for strings.
  • Value: A piece of data within a JavaScript program, such as a number (13), a string ("hello"), or a boolean (true). All values are made of bits but have a specific type.

2. Program Structure

  • Argument: A value passed to a function when it is called (invoked).
  • Binding (Variable): A name that refers to a value stored in the program's memory. Bindings allow programs to keep track of data. Created using let, const, or var.
  • Block: A sequence of statements grouped together within curly braces { }. JavaScript treats a block as a single statement.
  • break Statement: A keyword used to immediately exit the innermost enclosing loop (while, do, for) or switch statement.
  • case: A keyword used within a switch statement to define a specific value to compare against the switch expression. Execution jumps to the matching case.
  • Comment: Text included in program code (//... or /*...*/) that is ignored by the computer but serves as explanation for human readers.
  • Conditional Execution: Controlling whether a piece of code runs based on a condition being true or false, typically using if, else, and switch.
  • console.log: A function commonly available in JavaScript environments used to output values to a console, helpful for observing program behavior.
  • const: A keyword used to declare a constant binding, meaning the binding must be initialized with a value and cannot be reassigned to point to a different value later.
  • continue Statement: A keyword used inside a loop to skip the remainder of the current iteration and proceed to the next iteration.
  • Control Flow: The order in which statements are executed in a program. Control flow can be altered by loops, conditional statements, and function calls.
  • default: A keyword used within a switch statement to define the block of code to run if no case matches the switch expression's value.
  • do Loop: A control flow statement that creates a loop. The loop's body is executed once before the condition is checked, and then repeats as long as the condition remains true.
  • else: A keyword used in conjunction with if to specify a block of code that should be executed only when the if condition is false.
  • Environment: The collection of all bindings and their corresponding values that exist at a particular point in time during program execution.
  • Expression: A piece of code that produces a value (e.g., 22, "hello", 5 * 5, true, console.log("hi")).
  • for Loop: A control flow statement often used to loop a specific number of times. It typically includes an initialization part, a condition part (checked before each iteration), and an update part (executed after each iteration).
  • Function: A piece of program code wrapped in a value. It can be executed (called/invoked) by using parentheses (), possibly passing arguments, and may return a value.
  • Function Call (Invoking, Calling, Applying): The act of executing the code within a function.
  • if Statement: A keyword used for conditional execution. It executes the following statement or block only if the specified condition evaluates to true.
  • Indentation: The practice of using spaces or tabs to offset lines of code within blocks, making the program's structure easier for humans to read.
  • Infinite Loop: A loop whose condition never becomes false, causing the program to run forever (or until manually stopped). Often caused by errors in loop logic or missing break statements.
  • Keyword: A word that has a special meaning in the JavaScript language (like let, if, while, function) and cannot be used as a binding name.
  • let: A keyword used to declare a binding (variable) whose value can be reassigned later.
  • Loop: A control flow structure that allows a block of code to be executed repeatedly based on a condition.
  • Number Function: A standard built-in function that attempts to convert its argument into a number value.
  • Number.isNaN Function: A standard function that checks if a given value is NaN (Not a Number), returning true if it is, and false otherwise.
  • Return Value: The value that a function produces upon completion. A function call itself is an expression that evaluates to its return value.
  • Semicolon (;): A character used to optionally terminate statements in JavaScript. Recommended for clarity and avoiding certain parsing errors.
  • Shorthand Update Operators: Operators like +=, -=, *=, /=, ++, -- that provide a concise syntax for updating a binding's value based on its previous value.
  • Side Effect: An action performed by a statement or function that affects the program's state or interacts with the outside world (e.g., modifying a binding, printing to the console, showing a prompt).
  • Statement: A unit of code that expresses an action to be carried out. A JavaScript program consists of a sequence of statements.
  • switch Statement: A control flow statement that allows dispatching execution to different code blocks (case labels) based on the value of a single expression.
  • var: An older keyword (pre-2015) used to declare bindings. It has slightly different scoping rules compared to let and is less commonly used in modern JavaScript.
  • Variable: See Binding.
  • while Loop: A control flow statement that creates a loop. It repeatedly executes a statement or block as long as a given condition evaluates to true. The condition is checked before each iteration.
